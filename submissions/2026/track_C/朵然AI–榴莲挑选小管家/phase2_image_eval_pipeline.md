# 第二阶段：照片驱动的榴莲评测分析方案（CV + 数学计算 + LLM）

## 目标

在已有核心输入参数（重量、品种、形态、房数等）基础上，把“照片”作为主要信号源，构建一个可解释、可迭代、可量化的评测流程：

- 产出：出肉率区间/等级、质量风险点、解释文本、置信度
- 强约束：流程可拆分为可验证的 CV/数学指标，LLM 只做“融合与解释/补全”
- 可扩展：未来可替换为更强的本地 CV/轻量模型（或端上/端外混合），但数据结构与流程不推倒重来

## 输入与采集规范

### 输入参数（结构化）

- 必填：重量（kg）
- 推荐：品种（枚举）；形态（枚举）；房数（大房/小房）
- 可选：成熟度自报、是否开裂、气味等（先不作为核心）

### 照片（结构化）

采用 5 张固定角度（便于后续 CV 的几何一致性与标签稳定性）：

1. 上面（果柄/冠部）
2. 下面（底部）
3. 左侧
4. 右侧
5. 正面（尽量以“果柄在上”的方向定义）

采集规范建议（不需要一次性全上，先做“软提示 + 质量门禁”）：

- 背景尽量干净（浅色/单色优先）
- 主体尽量完整、不被裁切
- 焦点清晰、无明显抖动
- 光照均匀（避免强反光）

## 总体评测流程（优先级从高到低）

### P0：质量门禁（Quality Gate）

目标：决定“这组照片是否可用于分析”，不通过则只提示重拍，不进行后续计算。

建议指标（可全部在端上实现）：

- 主体识别：是否为榴莲（非榴莲直接拒绝）
- 裁切检测：主体边界是否被明显截断（可通过主体分割后与画面边缘接触比例判断）
- 清晰度：拉普拉斯方差/梯度能量阈值
- 曝光：均值亮度 + 方差（过暗/过曝）
- 反光：高亮区域占比 + 连通域形态（强高光）

输出：

- gateOk: Boolean
- issues: [non_durian, cropped, blurry, dark, overexposed, glare, too_far, other]

### P1：主体分割与标准化（Segmentation & Normalization）

目标：把每张照片归一到可比较的“主体坐标系”，为后续密度/角度等测量提供基础。

流程：

- 主体分割（优先可解释方案）
  - 规则/传统：边缘 + GrabCut + 形态学（对复杂背景鲁棒性一般）
  - 推荐：轻量分割模型（后续可训练/蒸馏；短期先用启发式）
- 关键几何量
  - 主体外接矩形、最小外接矩形（rotated rect）
  - 主轴方向（PCA on mask）
  - 归一化尺度：以主体长边/短边作为尺度单位（避免依赖真实厘米）

输出（每张图）：

- mask（主体二值图）
- bbox / rotated bbox
- majorAxisAngle（主轴角）
- scaleUnit（以像素为单位的归一尺度）

### P2：刺（spike）特征：密度 / 方向 / “高度代理”

目标：提取对“成熟度、皮厚/壳厚、品相”敏感且可解释的形态学特征。

#### 2.1 刺密度（Spike Density）

可行定义（无需真实物理尺度）：

- 在主体 mask 内，检测刺尖候选点数量 N
- 归一化面积 A（mask 面积，或按网格分块面积）
- 密度 d = N / A

实现路径：

- 先做纹理增强：高通/DoG/Laplacian
- 角点/局部极值检测：Harris/FAST/LoG blob
- 约束：只保留位于 mask 内、且局部梯度强度超过阈值的点
- 去重：非极大值抑制（NMS）+ 最小间距

输出：

- density_global（全局密度）
- density_map（网格分块密度，用于判断“密度是否均匀”）

#### 2.2 刺方向/角度分布（Orientation / Angle Distribution）

理论依据方向：可用梯度方向统计、结构张量（structure tensor）等经典纹理分析方法。

实现思路：

- 在 mask 内计算梯度（Sobel/Scharr）
- 对梯度方向做直方图（类似 HOG，但只聚焦刺纹理区域）
- 对每个网格块输出主方向与分散度（entropy/variance）

输出：

- orientation_hist（主方向直方图）
- orientation_entropy（方向分散度：越大表示越“乱/不一致”）
- orientation_main（主方向）

解释示例：

- “刺方向一致且分布集中” vs “方向分散、局部扭曲”

#### 2.3 刺“高度代理”（Height Proxy）

严格的真实高度需要深度信息/标定。端上可先做“代理量”，作为风险提示而非硬结论：

- 边缘粗糙度：主体轮廓的曲率/分形复杂度/周长-面积比（刺越长，轮廓更粗糙）
- 阴影线索（如采集能控制侧光）：刺影长度分布（需要拍摄提示配合）
- 单目深度（后续阶段）：用轻量深度估计网络获得相对深度起伏，提取“起伏幅度”

输出：

- boundary_roughness（轮廓粗糙度）
- depth_variation（可选，后续）

### P3：形态特征（Shape & Asymmetry）

目标：用几何特征支撑“形态（圆/荔枝形/歪尾/畸形）”并与用户输入互相校验。

特征（每张/多张融合）：

- 长宽比（aspect ratio）
- 圆度（4πA/P²）
- 对称性：mask 左右/上下镜像差异
- 主轴弯曲：拟合中心线的偏移（歪尾趋势）

输出：

- roundness
- aspect_ratio
- symmetry_score
- tail_offset_score

### P4：品种先验（Variety Priors）与参数融合

目标：品种已知时，使用品种的统计平均值/分布作为先验，避免纯视觉推断的漂移。

示例（后续用数据迭代）：

- 不同品种典型刺密度区间、皮厚倾向
- 不同品种在同等重量下的典型尺寸/体型范围

融合方式（建议从可解释开始）：

- 先验为每个特征提供“期望范围”，计算偏差 z-score 或分位数
- 把偏差作为风险/加权项，而不是直接替代观测
 
品种相关的先验资料库独立维护在：`doc/variety_priors.md`（用于 weight/yield/形态等范围先验与来源引用）。

### P5：LLM 介入点（谨慎使用）

LLM 的职责建议限定为：

- 把结构化特征转换为可读解释（“为什么是这个等级”）
- 在“特征冲突”时给出澄清问题（例如建议重拍某角度）
- 给出建议动作（继续拍/补充参数/查看结果）

不让 LLM 直接从原图做最终定量结论（除非能保证可控与一致性），更不直接让 LLM 覆盖硬门禁结果。

## 评分/权重（可落地的第一版）

定义最终评分为 0–100（或等级 1–5），由三部分组成：

1) 结构化输入（参数）得分：S_param
- 权重建议：20%
- 例如重量与房数一致性、品种已知加成、明显不合理值扣分

2) 视觉形态学得分：S_cv
- 权重建议：60%
- 由刺密度、方向一致性、轮廓粗糙度、对称性等组成

3) 质量门禁/置信度：C_quality
- 权重建议：20%（不直接加分，而是调低置信度、触发重拍）
- gate 不通过直接拒绝输出评测，只输出“重拍建议”

第一版融合可以用线性加权（可解释、易调参），后续再升级为学习型模型（回归/分类）。

## 数据结构（建议）

保存每次评测的“可复现记录”，便于离线验证与迭代：

- sessionId
- paramsSnapshot（重量/品种/形态/房数…）
- photos: [{angle, path, qualityGate, maskMeta, features}]
- score/level
- confidence
- explainText（由 LLM 生成）

## 迭代路线图（不为了复杂而复杂）

### 第 1 里程碑（最短闭环）

- 质量门禁完善（现有 Doran 检查 + 快速图像指标）
- 主体分割（先启发式，必要时再上轻量模型）
- 刺密度 + 方向直方图 + 轮廓粗糙度
- 简单线性权重融合 + 解释生成

### 第 2 里程碑（更强稳定性）

- 训练/蒸馏轻量分割模型（端上）
- 加入更稳的刺尖检测（局部极值 + 学习型校验）
- 多角度一致性检查（同一颗榴莲不同角度特征应在合理范围）

### 第 3 里程碑（更接近“高度/深度”）

- 引入相对深度估计（端上轻量 or 端外批处理）
- 刺高度代理升级（深度起伏 + 轮廓/阴影联合）

## 为什么这个方案“有理论依据”

核心思想是把图像问题拆成经典可验证模块：

- 质量门禁：清晰度/曝光/裁切检测属于标准图像质量评估范畴
- 纹理方向：梯度方向直方图、结构张量等是传统纹理/形状分析基石
- 密度估计：点过程密度（N/A）+ 分块统计是稳健的统计方案
- 形态几何：圆度、对称性、PCA 主轴属于经典形状描述符
- 先验融合：贝叶斯/统计先验思想（用品种分布约束观测）

LLM 只做“解释与流程调度”，把不可控成分限制在安全范围内。
