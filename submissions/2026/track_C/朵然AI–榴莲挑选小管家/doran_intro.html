Doran AI 端侧榴莲挑选 Agent

一个基于 Android 原生技术栈、Google ADK Kotlin、Gemma 4 2B Edge 和 LiteRT-LM 的本地智能体应用。 它引导用户拍摄五角度榴莲照片，补齐重量、房数、形态和品种，并在设备端完成照片质检、分析流程编排和购买建议生成。

Gemma 4 Edge + ADK Agent

模型导入、URL 下载、ADB push、CPU/GPU 后端回退、Agent-driven UI 和 P0-P5 Demo 工具链已经形成可运行闭环。

为什么值得做

榴莲购买是高客单价、高经验依赖的现场决策。Doran AI 把随手判断变成结构化采集和端侧推理，减少网络依赖，也保留用户照片和购买记录的隐私。

真实需求

消费者想知道这颗榴莲是否值得买，档口和团购卖家也需要更稳定的现场验货辅助。

端侧可用

核心推理在 Android 本机运行，适合市场、商超和产地等网络不稳定场景。

可扩展

五角度采集、状态机、报告 trace 和品种先验可以扩展到更多农产品质检场景。

作品亮点

这些能力共同构成了 Doran AI 的端侧体验：能在现场跑、能引导用户补信息，也能把分析过程和结果解释清楚。

技术架构

以 `AgentChatViewModel` 为状态中心，ADK Agent 负责把 Gemma 4 输出转成工具动作，LiteRT-LM 负责本地推理。

核心模块

UIJetpack Compose、动态聊天卡片、相机组件、表单组件、分析进度和报告卡片。
AgentGoogle ADK Kotlin、Native Function Calling、FunctionTool、本地 JSON 工具调用协议。
LLMGemma 4 2B Edge + LiteRT-LM Android，支持文本、视觉、音频接口和 CPU/GPU 回退。
DataDataStore 保存偏好和统计，JSON 文件保存会话、报告和 trace。

Demo 运行链路

1导入、下载或 ADB push `.litertlm` 模型。
2进入对话页，懒加载本地 Gemma 4 LiteRT-LM Engine。
3Agent 解析自然语言，更新重量、房数、形态和品种。
4CameraX 采集五角度照片，本地视觉模型做照片质检。
5P0-P5 Demo 工具链生成报告，保存历史并更新徽章/桌面组件。

P0-P5 分析管线

当前管线已经具备 ADK 工具链、进度事件、trace 和报告结构。其中 P1-P3 的真实 CV 计算仍是 Demo 代理值，后续可替换。

P0照片质量门禁照片数量、角度覆盖、风险项、参数完整性。
P1分割与归一化当前为 Demo 代理值，预留真实主体分割。
P2刺特征密度、方向一致性、高度代理，当前为 Demo。
P3形态几何体型、对称性、壳厚代理，当前为 Demo。
P4品种先验金枕、猫山王、黑刺、托曼尼、D24 等。
P5融合报告评分、Level、解释和购买建议。
