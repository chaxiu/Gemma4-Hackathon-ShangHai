# 5 分钟演示视频脚本底稿

本文件只作为后续视频线程的录制脚本。本次提交包不包含最终视频。

## 时间分配

| 时间 | 内容 |
|---|---|
| 0:00-0:30 | 痛点：孩子每天任务混杂，家长不想把系统变成控制台 |
| 0:30-1:10 | 评测驱动：先用 60 case 测 `gemma-4-26b-a4b-it` |
| 1:10-2:20 | 孩子端：输入、AI 反问、生成轻量计划 |
| 2:20-3:10 | 睡前复盘：孩子一次输入，系统结算与鼓励 |
| 3:10-4:10 | 家长端：真实完成、pending、判断依据、趋势 |
| 4:10-4:45 | Streamlit：RAG、trace、official/debug、评测证据 |
| 4:45-5:00 | 总结：本地模型 + memory + tools，保护隐私和减负 |

## 推荐孩子端输入

第一步：

```text
我今天4点半到家，有语文大小册和习字册，数学还有一点大小册和口头作业，英语有背诵和抄写。还有阅读和打卡题。我今天有点累，不知道先做什么。
```

AI 应出现 1-3 个反问，重点问数学剩余量、语文最费时部分、可用时间。

第二步：

```text
数学还剩大小册各一点，大概25分钟。语文大小册比较费时间，习字册可以后面做。晚饭前我能先做40分钟，今天有点累。
```

睡前复盘：

```text
数学大小册和口头作业做完了，英语背诵和抄写也完成了。语文大小册做完了，习字册还剩一点。阅读没做，打卡题也没做，今天有点累。
```

## 旁白要点

1. “我们没有把 Gemma 4 当万能聊天机器人，而是先做 60 case 能力边界评测。”
2. “评测显示它适合做候选 Planner，但时间、任务上限和安全语气必须由代码兜底。”
3. “孩子端不显示调试信息，只保留反问、计划、结算。”
4. “家长端不是加压工具，而是看真实完成和轻量纠偏。”
5. “Streamlit 是内部调试台，展示 RAG、trace、official/debug 隔离和评测证据。”
